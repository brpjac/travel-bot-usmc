# ForO 3000-52.1, Chapter 6 — T3 Database (verbatim)

## 1. Introduction (p. 6-1)

The T3 database is a web based asp.net application operating in real time with
the primary purpose of managing fiscal resources, exercises and the movement
of personnel, equipment and supplies. T3 is also the only method of scheduling
commercial travel outside of DTS. The database allows training and exercise
requirements to be matched with available financial resources in an organized
manner that permits Command oversight and approval of each event. There are
two types of events in T3; TEEP funded and Non-TEEP funded. Each TEEP funded
event must identify unit personnel and equipment requirements, modes of
transportation requested, fiscal requirements by fund category and total
amount of funds necessary to execute the event.

## 2. T3 Access (p. 6-1)

a. Unit training personnel must possess a valid Common Access Card (CAC) and
the appropriate access permissions to utilize the T3 database.

b. The T3 database can be accessed through the following URL:
https://t3.mcdsus.mcds.usmc.mil/.

c. Permissions can be assigned by the MARFORRES or MSC T3 Managers.
Permissions are granted for appropriate level based on a need for access to
the T3 data base. Current types of permissions: (1) User permission — allows
the user to create and submit both TEEP and Non-TEEP funded requests. (2) Read
only permission — restricts user only to be able to view data in the T3
database. (3) Inactive account — no access. (4) DMO permission — provides
special access for MARFORRES G4 personnel. (5) T3 Manager permission — allows
MARFORRES and MSC T3 managers to create T3 Events as well as create and submit
TEEP/Non-TEEP event requests. (6) Administrative permission — provides
software engineer rights, only.

## 3. T3 Event/Request (pp. 6-2 to 6-3)

a. The T3 database contains information that provides detailed instructions
for training personnel and T3 managers to reference and assist them as they
navigate the T3 database. The information is contained in the "T3 Video
Training" tab within the T3 database. The information covers both submitting
TEEP/Non-TEEP funded requests and creating TEEP/Non-TEEP funded T3 events.

b. Creating a T3 Event. Only the MARFORRES or MSC T3 managers can create T3
events. Fundamental elements T3 managers must understand prior to creating an
event:

(1) **Level of Event.** The level of the event will determine whether the TEEP
event is created by the MARFORRES T3 Manager or the MSC T3 managers.
(a) Chairman of the Joint Chiefs of Staff (CJCS) exercises or events are those
sponsored by the CJCS and will only be created by the MARFORRES T3 Manager.
CJCS events use Joint Exercise Program funding such as SIF and CTP. CJCS
exercises will be identified as a Force level exercise by using an F-level
event number.
(b) A Force level exercise/event is one in support of DOD, Service Level
Training Exercises, MARFORRES directed exercises, post-exercise maintenance
(PEM) or any event involving more than one MSC. Creation of these exercises in
the T3 database will be by the MARFORRES T3 Manager. Force exercises will be
identified as a Force level exercise by using an F-level event number.
(c) Non-CJCS OCONUS exercises/events are in support of any exercise executing
outside the continental US regardless of scope of the exercise or the number
of participating MSCs. Creation of these exercises will be by the MARFORRES T3
Manager. Non-CJCS OCONUS exercises will be identified as a Force level
exercise by using an F-level event number.
(d) Innovative Readiness Training (IRT) exercises/events are sponsored by
OSD/RA and provide training for our forces while supporting the community. IRT
events are created in the T3 database by the MARFORRES T3 Manager. These TEEP
events use IRT funds from OSD RA in combination with other TEEP funds. IRT
events will be identified as a Force level exercise by using an F-level event
number.
(e) Major Subordinate Command (MSC) exercises are those exercises directed by
MSC Commanding Generals or their subordinate Commanders within the MSC. The
MSC T3 managers are authorized to create these exercises/events within the T3
database. MSC exercises/events will be identified as using an event number
corresponding to the MSC creating the TEEP exercise/event. Fourth Marine
Division will be identified with a number beginning with a D, Fourth Marine
Logistics Group with an M, Fourth Marine Aircraft Wing with a W, and Force
Headquarters Group with a G.

(2) **Special Interest Code (SIC).** SICs are three place alpha numeric codes
use to identify the T3 Event, e.g. African Lion SIC = DZ0. It is imperative
that the proper SIC is used consistently with the exercise it is associated
with to ensure proper management of funds resourced for the exercise.

(3) **Manning.** Close attention must be paid to the manning
documents/personnel requirements to ensure accurate numbers are provided to
permit proper estimation of funds required for an exercise/event. As changes
occur, the T3 database must be updated. This will ensure the proper amount of
fiscal resources is allocated for the exercise.

(4) **Expense Categories.** T3 managers and training personnel need to have a
working knowledge of the multiple types of funding and which exercises and
events the funding applies to. Accurate data and the correct categories of
funds facilitates approval of T3 requests and the creation of TEEP/Non-TEEP
funded events.
