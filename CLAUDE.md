# MIU Travel Regulation Bot

RAG chatbot for USMC reserve Marines to query travel regulations (JTR, MCRAMM, MARADMINs).

## Stack

- **LLM:** Google Gemini 2.5 Flash (free tier)
- **Embeddings:** all-MiniLM-L6-v2 via sentence-transformers (local, no API)
- **Vector store:** FAISS (local, file-based)
- **Framework:** LangChain (Python)
- **Frontend:** Streamlit
- **Hosting:** Streamlit Community Cloud

## Project Structure

```
app.py                  # Streamlit app (frontend + RAG pipeline)
scripts/ingest.py       # PDF/text ingestion → FAISS vector store
docs/                   # Source documents (PDFs and .txt files)
docs/metadata.json      # Document metadata registry
vectorstore/            # FAISS index (generated by ingest.py)
logs/                   # Token usage logs (generated at runtime)
```

## How to Run

```bash
export GOOGLE_API_KEY="your-key-here"
python scripts/ingest.py          # Build the vector store
streamlit run app.py              # Start the app
```

## Adding a New Document

1. Drop the PDF (or .txt file) into `docs/`
2. Add an entry to `docs/metadata.json`:
   ```json
   "MARADMIN 123-26.pdf": {
     "source_doc": "MARADMIN 123/26",
     "doc_type": "maradmin",
     "date": "2026-01-15"
   }
   ```
   Valid `doc_type` values: `standing_regulation`, `standing_order`, `maradmin`, `unit_reference`
3. Run incremental ingestion:
   ```bash
   python scripts/ingest.py --incremental
   ```
4. Push to GitHub — Streamlit Cloud redeploys automatically

If the document is a visual/flowchart PDF that won't extract well, create a `.txt` version
(see `docs/orders-type-matrix.txt` as an example) and register that in metadata.json.

## Full Re-index

To rebuild the entire vector store from scratch:
```bash
python scripts/ingest.py
```

## Token Usage Logs

Token usage is logged to `logs/token_usage.csv` with columns:
`timestamp, question_length, prompt_tokens, response_tokens, total_tokens`

Use this to monitor whether conversation context is worth the token cost.
The sidebar toggle "Use conversation context" can be turned off if tokens are burning too fast.
